# 📌 MVP - [Aprendizado-por-Projeto-Integrado-API2-]

## 🎯 Objetivo do MVP
> Descrever de forma clara qual é o propósito do MVP:  
- Qual problema resolve?
A solução resolve a dificuldade de visualização e organização das informações do sistema FacLog, onde o dashboard do Power BI não possuía identidade visual da marca e o banco de dados apresentava dados fragmentados e redundantes, dificultando análises precisas.

- Qual hipótese será validada?
A hipótese é que uma interface mais apresentável e com dados consolidados no banco aumenta a clareza das informações e facilita a tomada de decisão dos usuários que utilizam o painel.

- Qual valor será entregue ao usuário final?  
O usuário terá acesso a um painel Power BI com identidade visual da marca FacLog, navegação mais limpa e dados de itens unificados, permitindo análises mais confiáveis e uma experiência mais profissional.

---

## 📝 Descrição da Solução
> Breve explicação do que será desenvolvido e entregue nesta etapa.
 
Nesta sprint foram desenvolvidas melhorias visuais no painel Power BI e reestruturação do banco de dados para consolidação de itens semelhantes.

Funcionalidades principais incluídas:
*Capa personalizada no Power BI com a logo da FacLog
*Atualização de tipografia e formatação textual dos tópicos
*Remoção de botões não utilizados para limpeza da interface
*Congregação de itens similares no banco de dados (ex: tipos de álcool unificados em uma única categoria)

Limitações conhecidas:
A congregação de itens ainda pode não cobrir todos os casos de redundância existentes no banco
Melhorias visuais são iniciais e podem precisar de refinamento com base no feedback dos usuários

Escopo reduzido:
Focado exclusivamente na apresentação visual do painel e na organização básica dos dados, sem adição de novas funcionalidades analíticas.

## 📊 Critérios de Aceitação
- O MVP deve permitir que o usuário visualize a capa do Power BI com a logo da FacLog corretamente exibida
- O sistema deve apresentar os tópicos com tipografia padronizada e legível
- O banco de dados deve registrar itens similares sob uma única categoria consolidada (ex: todos os tipos de álcool agrupados como "Álcool")
- A interface do painel não deve exibir botões sem funcionalidade atribuída
- Métricas coletadas: número de categorias consolidadas no banco, número de elementos visuais removidos ou ajustados 

---

## 📈 Métricas de Validação
- Número de usuários que acessaram e navegaram pelo painel após as alterações
- Feedback qualitativo sobre a identidade visual (a marca FacLog está clara e reconhecível?)
- Redução de registros redundantes no banco após a consolidação de itens
- Indicadores de negócio: redução de erros em relatórios causados por itens duplicados, melhora na percepção de profissionalismo do painel
